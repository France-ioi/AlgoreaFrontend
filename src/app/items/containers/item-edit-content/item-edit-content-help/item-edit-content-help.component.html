The description is rendered inside a sandboxed iframe. You can use HTML, CSS and JavaScript freely, but author code cannot reach the rest of the application: it has no access to the parent page, cookies, local storage, or the user's session.

What is allowed

  • HTML: most tags including headings, paragraphs, lists, tables, images, blockquotes, code blocks…
  • CSS: inline style attributes and <style> blocks. Author styles override the default Algorea typography because they are loaded after the base stylesheet.
  • JavaScript: inline <script> blocks run inside the sandbox. Use them for interactive content (animations, small widgets, hint reveals…).
  • Images: any reachable HTTPS URL, or inline data: URIs.

What is not allowed

  • Reading or writing parent-page state (DOM, cookies, localStorage, the user's session…). The sandbox has an opaque origin, so any such access throws.
  • Replacing the application page. window.top navigation is blocked by the sandbox; clicks on links with a regular href are routed through the parent instead (see below).
  • javascript:, data:, file:, blob:, mailto:, tel:, ftp:… as link targets. They are silently dropped — only http(s) external URLs are accepted.
  • Loading custom fonts from the parent CSS. If you need a specific font face, declare an @font-face rule inside your description.

Creating links

Because the iframe cannot navigate the application by itself, every link must tell the parent what to do. Use one of the following anchor patterns (label text inside the examples is illustrative — replace it with your own).

Navigate to another item (by ID)

<a data-item-id="1234567890">link label</a>

Add data-child to navigate as a child of the currently displayed item (useful for drill-down inside a chapter): 

<a data-item-id="1234567890" data-child>link label</a>

Navigate to an external URL

Either use a regular href or the explicit data-url attribute. External links always open in a new tab. 

<a href="https://example.com">link label</a>
<a data-url="https://example.com">link label</a>

Only absolute http(s) URLs are accepted. Relative paths (e.g. /groups/42) cannot be used to navigate inside the application — use data-item-id instead.

Jump to a section inside the description

Hash anchors work as expected — the parent scrolls to the target. Targets are resolved by id first, then by the legacy name attribute. 

<a href="#summary">link label</a>
…
<h2 id="summary">Summary</h2>
Advanced: scripted navigation and custom height

Author scripts can reach the parent through three postMessage events. They are the same channel the anchor handlers use, so use them when you need scripted behavior (e.g. opening an item from a button click).

Trigger a navigation from a script

parent.postMessage({
  type: 'alg.navigate',
  data: { itemId: '1234567890', child: true },
}, '*');

parent.postMessage({
  type: 'alg.navigate',
  data: { url: 'https://example.com' },
}, '*');

Scroll the parent to a position inside the iframe

The iframe itself never scrolls (the parent owns the height), so use this message instead of window.scrollTo when you want to move the user to a specific Y offset (in CSS pixels, measured from the top of the iframe document). 

parent.postMessage({
  type: 'alg.scrollIntoView',
  data: { offset: 0 },
}, '*');

Override the rendered height

The iframe height is measured automatically with a ResizeObserver, so you usually do not need to do anything. Send this message only if you need to force a specific height in CSS pixels (e.g. while a heavy widget loads). 

parent.postMessage({
  type: 'alg.updateDisplay',
  data: { height: 600 },
}, '*');

Messages with an unknown type, an invalid payload, or a non-http(s) URL are silently dropped — there is no error feedback in the iframe.
